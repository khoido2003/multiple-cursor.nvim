1. INTRODUCTION                                 *multiple-cursor-introduction*

Multiple Cursor is a lightweight, independent Neovim plugin that provides
multiple cursor functionality with intelligent word matching and easy
selection/skip workflow.

Features:
- Word Matching: Find all matching occurrences of word under cursor
- Select/Skip Workflow: Easy navigation to include or exclude matches
- Synchronized Editing: Edit all selected locations simultaneously
- User-Configurable Keymaps: Full customization of all keybindings
- Zero Dependencies: Standalone plugin using only Neovim's built-in APIs

==============================================================================
2. INSTALLATION                                 *multiple-cursor-installation*

Using lazy.nvim: >lua
  {
    "khoido2003/multiple-cursor.nvim",
    config = function()
      require("multiple-cursor").setup()
    end,
  }
<

Using packer.nvim: >lua
  use {
    "khoido2003/multiple-cursor.nvim",
    config = function()
      require("multiple-cursor").setup()
    end,
  }
<

3. QUICK START                                   *multiple-cursor-quickstart*

1. Place your cursor on any word
2. Press <C-n> to select it (cursor jumps to next match)
3. Press <C-n> to select next match, or <C-x> to skip/remove
4. Use <C-j>/<C-k> to navigate between matches
5. Press `c` to change all selected words, or `d` to delete them
6. Press <Esc> to exit

==============================================================================
4. KEYMAPS                                         *multiple-cursor-keymaps*

Default keymaps when multi-cursor mode is active:

                                                    *multiple-cursor-<C-n>*
<C-n>                   Add cursor to match under current cursor position.
                        If no session active, starts one on current word.

                                                    *multiple-cursor-<C-x>*
<C-x>                   Skip or remove cursor from match under current position.
                        Moves match to 'skipped' list.

                                                    *multiple-cursor-<C-j>*
<C-j>                   Jump to next match occurrence.

                                                    *multiple-cursor-<C-k>*
<C-k>                   Jump to previous match occurrence.

                                                    *multiple-cursor-<C-a>*
<C-a>                   Select all remaining matches at once.

                                                    *multiple-cursor-<Esc>*
<Esc>                   Exit multi-cursor mode and clear all cursors.

                                                    *multiple-cursor-c*
c                       Change all selected words.
                        Text you type will appear at all cursor positions.

                                                    *multiple-cursor-d*
d                       Delete all selected words.

                                                    *multiple-cursor-i*
i                       Enter insert mode at all selected positions.

5. CONFIGURATION                               *multiple-cursor-configuration*

Call the setup function with a configuration table: >lua

    require("multiple-cursor").setup({
    -- Keymaps (set to false to disable)
    keymaps = {
      start_next = "<C-n>",      -- Start or add next match
      skip = "<C-x>",            -- Skip/remove current match
      next_match = "<C-j>",      -- Jump to next match
      prev_match = "<C-k>",      -- Jump to previous match
      select_all = "<C-a>",      -- Select all matches
      exit = "<Esc>",            -- Exit multi-cursor mode
    },

    -- Highlight groups
    highlights = {
      cursor = "MultipleCursor",
      match = "MultipleCursorMatch",
    },

    -- Behavior
    match_whole_word = true,    -- Only match whole words
    case_sensitive = true,      -- Case sensitive matching
  })
<

                                           *multiple-cursor-config-keymaps*
keymaps ~
  Table of keymaps. Each can be a string (the keymap) or `false` to disable.

                                        *multiple-cursor-config-highlights*
highlights ~
  Table of highlight group names used by the plugin.

                                    *multiple-cursor-config-match_whole_word*
match_whole_word ~
  Boolean. When true, only matches whole words (default: true).
  When false, matches substrings as well.

                                     *multiple-cursor-config-case_sensitive*
case_sensitive ~
  Boolean. When true, matching is case-sensitive (default: true).
  When false, matching ignores case.

6. COMMANDS                                       *multiple-cursor-commands*

                                                    *:MultipleCursorStart*
:MultipleCursorStart    Start multi-cursor mode on word under cursor.

                                                   *:MultipleCursorClear*
:MultipleCursorClear    Clear all cursors and exit multi-cursor mode.

                                               *:MultipleCursorSelectAll*
:MultipleCursorSelectAll
                        Select all remaining matches.

==============================================================================
7. API                                                 *multiple-cursor-api*

The plugin exposes the following Lua API: >lua

  local mc = require("multiple-cursor")

  -- Setup the plugin (required)
  mc.setup(opts)

  -- Start multi-cursor mode on word under cursor
  mc.start()

  -- Check if multi-cursor mode is active
  mc.is_active()  -- returns boolean

  -- Exit multi-cursor mode
  mc.exit()
<

==============================================================================
8. HIGHLIGHTS                                     *multiple-cursor-highlights*

The plugin defines the following highlight groups:

                                                        *hl-MultipleCursor*
MultipleCursor          Used for selected cursor positions.
                        Default: green background with bold text.

                                                    *hl-MultipleCursorMatch*
MultipleCursorMatch     Used for unselected matches.
                        Default: dim yellow background with underline.

                                                  *hl-MultipleCursorSkipped*
MultipleCursorSkipped   Used for skipped matches.
                        Default: dim red background with strikethrough.

You can override these via :highlight command or checks logic. Note that
default colors are forced on ColorScheme change to ensure visibility.
